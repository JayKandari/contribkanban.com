How to contribute

First, checkout the contributing doc. The best contributions are ideas and concepts to help make working on Drupal contrib (and core!) easier.

Project boards

Projects (distributions, themes, modules) can be added from the browse page - simply enter the project's machine name! Project boards can be customized to fit different needs. For example, Panelier and Commerce Reports focus on a parent Plan issue and group by child issues. Drupal Commerce and Panopoly only show issues tagged with "sprint".

To customize a board, either open an issue labeled "project board request" or open a pull request to add/change it's configuration. More info on custom project boards

Sprint boards

Sprint boards are based off of Drupal.org issue tags. Currently there isn't a UI for adding sprint boards (#9). Requests can be made in the project's issue queue labeled "sprint board request." Sprint board layouts push the focus on working on in-progress issues before starting new ones (Needs Review and Needs Work before Active.)

Issues are missing?

Currently pagination is not supported in this app (API supports it) solely on the fact that I don't want to bog down the API with irrelevant requests. A single query returns 100 nodes, which should suffice for the most part.

This app instead requests the issues to be sorted by most recently modified. That means fresh issues will always display.
